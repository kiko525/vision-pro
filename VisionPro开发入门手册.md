# VisionPro 开发入门手册

本文档面向第一次接手本项目的本科生，目标是帮助大家先把项目跑起来、看懂、能定位问题，再进入具体开发。

如果需要更完整的背景说明，请配合阅读：

- `VisionPro需求说明文档.md`

---

## 1. 这是什么项目

这是一个基于 Web 技术的 3D 模型展示系统，支持以下设备和场景：

- 普通 PC 浏览器
- 移动端浏览器
- Apple Vision Pro Safari

系统当前有三种模式：

1. 普通浏览模式
2. AR/MR 模式
3. VR 模式

系统还包含一个管理后台，用于：

- 上传模型
- 上传缩略图
- 上传背景图
- 编辑模型信息
- 删除模型

---

## 2. 你接手时要先知道什么

### 2.1 这不是原生 Vision Pro App

本项目不是 visionOS 原生应用，而是一个 WebXR 项目。  
Vision Pro 访问本项目时，本质上是：

- 通过 `HTTPS` 打开网页
- 通过 `WebXR` 进入 `immersive-vr` 或 `immersive-ar`
- 通过 `WebSocket` 把前端日志传回服务端终端

### 2.2 当前项目能用，但还不够稳定

当前项目已经具备基本功能，但也要明确：

- 架构比较简单
- 模块边界不够清晰
- 有些功能已经可用，但不够稳定
- 有些能力已经有雏形，但还没形成闭环

所以后续工作不只是“继续加功能”，还包括：

- 把项目升级为更稳定的工程结构
- 把普通浏览模式的视觉表现向已有 demo 对齐

### 2.3 不要一上来就改 XR

建议顺序是：

1. 先把普通浏览模式看懂
2. 再把后台和数据流看懂
3. 再处理 Vision Pro 访问链路
4. 最后再改 `js/vr-module.js`

## 3. 你需要先掌握的最少概念

### 3.1 Three.js

本项目里的普通 3D 显示主要依赖 Three.js，主要负责：

- 场景 `Scene`
- 相机 `Camera`
- 渲染器 `Renderer`
- 灯光 `Light`
- 模型加载器 `Loader`
- 普通模式下的鼠标控制 `OrbitControls`

### 3.2 WebXR

本项目里的 XR 能力主要依赖 WebXR，重点是两个会话类型：

- `immersive-vr`
- `immersive-ar`

Vision Pro 的 WebXR 交互和普通教程里常见的“控制器射线”不完全一样，这也是后续难点之一。

### 3.3 Gaussian Splatting

除了普通网格模型，本项目还支持高斯泼溅模型。常见格式：

- `ply`
- `splat`
- `ksplat`
- `spz`

普通模式和 XR 模式对这类模型的加载方式并不一样，这一点必须记住。

## 4. 项目目录怎么读

建议先从下面这些文件开始看：

### 4.1 前台入口

- `index.html`
- `js/app.js`

这两个文件决定：

- 页面长什么样
- 模型怎么切换
- 普通模式怎么加载模型
- VR/MR 按钮怎么初始化

### 4.2 XR 入口

- `js/vr-module.js`

这是整个项目最复杂的文件，负责：

- 进入 VR/MR
- 画廊模式
- 详情模式
- Vision Pro 手势交互
- XR 中的高斯模型渲染

### 4.3 高斯模型普通模式入口

- `js/plyViewer.js`

这个文件负责普通浏览模式下的高斯模型显示。

### 4.4 后端与数据

- `server.js`
- `data/models.json`
- `data/backgrounds.json`

这里负责：

- HTTPS 服务
- API
- 文件上传
- 数据存储
- PLY 自动转换

### 4.5 管理后台

- `admin/index.html`
- `admin/js/admin.js`

这里负责模型管理和背景图管理。

## 5. 本地启动步骤

### 5.1 安装依赖

```bash
npm install
```

### 5.2 生成 HTTPS 证书

推荐方式：

```bash
npm run cert
```

备用方式：

```bash
npm run cert:selfsigned
```

注意：

- 推荐使用 `mkcert`
- 自签名证书只适合临时测试

### 5.3 启动项目

```bash
npm run dev
```

或者：

```bash
npm start
```

### 5.4 访问地址

本机：

- `https://localhost:3001`

局域网设备访问：

- `https://你的电脑IP:3001`

后台：

- `https://你的电脑IP:3001/admin/index.html`

## 6. Vision Pro 调试前必须知道的事

### 6.1 HTTPS 是硬要求

如果不是 HTTPS，Vision Pro 上的 WebXR 往往无法正常工作。

### 6.2 证书必须可信

你需要把下面的证书安装到 Vision Pro 上：

- `cert/rootCA.cer`

并在设备上启用完全信任。

### 6.3 证书里必须有当前访问 IP

Vision Pro 不是访问 `localhost`，通常访问的是你电脑在局域网里的 IP。  
所以：

- `cert-config.json` 里的 IP 要对
- 证书重新生成后才能生效

### 6.4 Vision Pro 调试时先看终端

项目带了 WebSocket 日志桥。  
很多前端日志会直接回传到服务端终端，所以：

- 不要只看页面现象
- 先看终端日志

## 7. 三种模式快速理解

### 7.1 普通浏览模式

入口：

- 页面默认进入

作用：

- 浏览模型
- 切换缩略图
- 查看名称和描述
- 重置视角
- 全屏

特点：

- 普通网格模型走 Three.js
- 高斯模型走 `plyViewer.js`

### 7.2 VR 模式

入口：

- 点击页面上的 `VR` 按钮

作用：

- 进入沉浸式空间画廊
- 选择模型进入详情
- 单手移动
- 双手缩放旋转

特点：

- 使用 `immersive-vr`
- 使用 `js/vr-module.js`

### 7.3 AR/MR 模式

入口：

- 点击页面上的 `MR` 按钮

作用：

- 在现实背景中叠加虚拟模型

特点：

- 使用 `immersive-ar`
- 现在还是基础版本
- 还不是完整的“点选平面后放置模型”的 AR 系统

## 8. 模型格式怎么理解

### 8.1 常规模型

当前代码实际支持：

- `glb`
- `gltf`
- `obj`
- `stl`
- `fbx`

最推荐：

- `glb`

### 8.2 高斯模型

当前支持：

- `ply`
- `splat`
- `ksplat`
- `spz`

最推荐：

- `ksplat`

### 8.3 当前要注意的现实问题

虽然前台代码支持 `obj/stl/fbx`，但后台上传入口目前没有把这些格式开放出来。  
所以你要区分：

- “代码支持”
- “后台当前可上传”

这两件事现在还不是完全一致的。

## 10. 当前项目的两个新增目标

### 10.1 架构升级目标

当前项目可以运行，但架构偏简单。后续应把以下方向作为明确目标：

- 更清晰的前后端模块边界
- 更稳定的渲染逻辑组织方式
- 更易维护的资源管理方式
- 更适合多人协作的工程结构

### 10.2 Demo 对齐目标

你们已经有一个“普通浏览模式下模型粒子加载效果”的 demo 项目。  
这个 demo 后续应作为普通浏览模式效果优化的重要参考。

学生接手时要明确：

- 这不是独立无关的 demo
- 它应该成为本项目普通浏览模式的一个对齐目标

重点观察 demo 中的内容：

- 粒子加载过渡
- 视觉反馈
- 模型出现方式
- 切换时的表现

## 11. 调试时最常见的坑

### 11.1 页面打不开，不一定是代码错

优先检查：

- 证书
- IP
- HTTPS
- 端口

### 11.2 普通模式能显示，不代表 XR 也能显示

因为：

- 普通模式和 XR 模式对高斯模型走的是不同渲染路径

### 11.3 AR/MR 不是透明背景就算做完

还要考虑：

- 地面参考系
- 模型位置稳定性
- 阴影
- 遮挡

### 11.4 不要在没搞懂状态机前直接改 XR

`js/vr-module.js` 虽然很长，但核心状态其实不多：

- `GALLERY`
- `DETAIL`
- session start
- session end

先画流程图，再改代码。

## 12. 你第一周应该做什么

### 第一步

把项目跑起来：

- 安装依赖
- 生成证书
- 启动服务
- 打开前台和后台

### 第二步

上传一个 `GLB` 模型，确认：

- 前台可显示
- 后台可管理
- 切换功能正常

### 第三步

上传一个 `PLY` 或 `KSPLAT` 模型，确认：

- 普通模式高斯模型能显示
- 模型切换后不会崩

### 第四步

在真正改 XR 之前，至少回答清楚下面几个问题：

- 普通模式怎么加载 `GLB`
- 普通模式怎么加载 `PLY/KSPLAT`
- `server.js` 怎么读写模型数据
- 后台上传后，数据写到哪里
