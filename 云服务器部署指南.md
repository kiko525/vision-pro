
## 🚀 快速部署（3 步完成）

### 第 1 步：购买云服务器

**国内**：
- [阿里云](https://www.aliyun.com/) - 稳定，学生优惠
- [腾讯云](https://cloud.tencent.com/) - 便宜，新用户优惠

**国外**：
- [AWS](https://aws.amazon.com/) - 全球最大
- [DigitalOcean](https://www.digitalocean.com/) - 简单易用
- [Vultr](https://www.vultr.com/) - 价格便宜

#### 推荐配置

**最低配置**（测试用）：
```
CPU: 1核
内存: 2GB
硬盘: 40GB
带宽: 1Mbps
系统: Ubuntu 20.04
价格: ¥60-80/月
```

**推荐配置**（生产用）：
```
CPU: 2核
内存: 4GB
硬盘: 60GB
带宽: 3-5Mbps

```

#### 购买注意事项

✅ **必须选择**：
- 公网 IP（必须！）
- Ubuntu 20.04 或更高版本
- 按量付费或包年包月

❌ **不要选择**：
- 仅内网 IP
- Windows Server（配置复杂）
- 过低配置（1核1G 可能不够）

### 第 2 步：配置服务器

#### 2.1 连接到服务器

```bash
# 使用 SSH 连接（替换为你的服务器 IP）
ssh root@123.45.67.89

# 首次连接会提示保存指纹，输入 yes
# 然后输入密码
```

**Windows 用户**：
- 使用 PuTTY 或 Windows Terminal
- 或使用 VS Code 的 Remote SSH 插件

#### 2.2 安装 Node.js

```bash
# 更新系统
sudo apt update
sudo apt upgrade -y

# 安装 Node.js 18.x
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node --version  # 应该显示 v18.x.x
npm --version   # 应该显示 9.x.x
```

#### 2.3 安装 PM2（进程管理器）

```bash
# 全局安装 PM2
sudo npm install -g pm2

# 验证安装
pm2 --version
```

#### 2.4 配置防火墙

```bash
# 安装 UFW（如果没有）
sudo apt install ufw

# 允许 SSH（重要！否则会断开连接）
sudo ufw allow 22

# 允许应用端口
sudo ufw allow 3001

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

### 第 3 步：部署应用

#### 3.1 在本地配置证书

编辑 `cert-config.json`，添加服务器 IP：

```json
{
  "certificate": {
    "subjectAltName": {
      "ip": [
        "127.0.0.1",
        "10.100.36.40",      // 本地 IP
        "123.45.67.89"       // 服务器公网 IP（添加这行）
      ]
    }
  }
}
```

生成证书：
```bash
npm run cert:force
```

#### 3.2 上传代码到服务器

**方式 A：使用 SCP**

```bash
# 在本地电脑上执行
scp -r ./* root@123.45.67.89:/root/3d-model-viewer/
```

**方式 B：使用 Git**

```bash
# 在服务器上执行
cd /root
git clone https://your-repo.git 3d-model-viewer
cd 3d-model-viewer
```

**方式 C：使用 FTP 工具**
- FileZilla
- WinSCP
- Cyberduck

#### 3.3 安装依赖并启动

```bash
# SSH 连接到服务器
ssh root@123.45.67.89

# 进入项目目录
cd /root/3d-model-viewer

# 安装依赖
npm install

# 使用 PM2 启动
pm2 start server.js --name "3d-model-viewer"

# 设置开机自启
pm2 startup
pm2 save

# 查看状态
pm2 status
pm2 logs 3d-model-viewer
```

#### 3.4 测试访问

在 Vision Pro 或任何设备上访问：
```
https://123.45.67.89:3001
```

✅ **成功！** 现在可以随时随地访问了！

## 🌐 使用域名（推荐）

### 为什么使用域名？

**使用 IP**：
- ❌ 难记忆：`https://123.45.67.89:3001`
- ❌ 证书警告
- ❌ 不专业

**使用域名**：
- ✅ 易记忆：`https://3dmodel.com`
- ✅ 无证书警告（Let's Encrypt）
- ✅ 更专业

### 步骤 1：购买域名

**推荐域名商**：
- [阿里云](https://wanwang.aliyun.com/) - ¥50-100/年
- [腾讯云](https://dnspod.cloud.tencent.com/) - ¥50-100/年
- [GoDaddy](https://www.godaddy.com/) - $10-20/年
- [Namecheap](https://www.namecheap.com/) - $10-20/年

**域名选择建议**：
- 简短易记
- 与项目相关
- 优先选择 `.com`

### 步骤 2：域名解析

在域名管理后台添加 A 记录：

```
类型: A
主机记录: @
记录值: 123.45.67.89（你的服务器 IP）
TTL: 600
```

如果要支持 `www`：
```
类型: A
主机记录: www
记录值: 123.45.67.89
TTL: 600
```

**等待生效**：通常 10 分钟到 1 小时

**测试解析**：
```bash
ping 3dmodel.com
# 应该显示你的服务器 IP
```

### 步骤 3：安装 Let's Encrypt 证书

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 安装 Nginx
sudo apt install nginx -y

# 获取证书
sudo certbot certonly --standalone -d 3dmodel.com -d www.3dmodel.com

# 按提示输入邮箱
# 同意服务条款
```

### 步骤 4：配置 Nginx

创建配置文件：
```bash
sudo nano /etc/nginx/sites-available/3dmodel
```

添加以下内容：
```nginx
# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name 3dmodel.com www.3dmodel.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name 3dmodel.com www.3dmodel.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/3dmodel.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/3dmodel.com/privkey.pem;

    # SSL 优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 反向代理到 Node.js
    location / {
        proxy_pass https://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|glb|gltf|ply|splat)$ {
        proxy_pass https://localhost:3001;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

启用配置：
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/3dmodel /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx

# 设置开机自启
sudo systemctl enable nginx
```

### 步骤 5：设置证书自动续期

```bash
# 测试续期
sudo certbot renew --dry-run

# 添加定时任务
sudo crontab -e

# 选择编辑器（推荐 nano）
# 添加以下行（每天凌晨 2 点检查续期）
0 2 * * * certbot renew --quiet --post-hook "systemctl reload nginx"
```

### 步骤 6：访问测试

在 Vision Pro 上访问：
```
https://3dmodel.com
```

✅ **完美！** 无证书警告，随时随地访问！

## 🐛 常见问题

### Q1: 无法连接到服务器

**检查**：
```bash
# 测试连接
ping 123.45.67.89

# 测试端口
telnet 123.45.67.89 3001
```

**可能原因**：
- 防火墙未开放端口
- 服务器安全组未配置
- 应用未启动

### Q2: 证书警告

**原因**：使用自签名证书

**解决**：
- 使用域名 + Let's Encrypt 证书
- 或在 Vision Pro 上安装证书

### Q3: 访问很慢

**优化**：
1. 升级服务器带宽
2. 使用 CDN 加速
3. 启用 Gzip 压缩
4. 优化模型文件大小

### Q4: 应用崩溃

**查看日志**：
```bash
pm2 logs 3d-model-viewer
```

**常见原因**：
- 内存不足（升级配置）
- 端口被占用
- 依赖未安装

## 🎯 最佳实践

### 安全建议

1. **修改 SSH 端口**
   ```bash
   sudo nano /etc/ssh/sshd_config
   # 修改 Port 22 为其他端口
   sudo systemctl restart sshd
   ```

2. **禁用 root 登录**
   ```bash
   # 创建新用户
   sudo adduser admin
   sudo usermod -aG sudo admin
   
   # 禁用 root
   sudo nano /etc/ssh/sshd_config
   # 设置 PermitRootLogin no
   ```

3. **配置防火墙**
   ```bash
   # 只开放必要端口
   sudo ufw default deny incoming
   sudo ufw default allow outgoing
   sudo ufw allow 22
   sudo ufw allow 80
   sudo ufw allow 443
   ```

### 性能优化

1. **启用 Gzip 压缩**
   ```nginx
   gzip on;
   gzip_types text/plain text/css application/json application/javascript;
   gzip_min_length 1000;
   ```

2. **配置缓存**
   ```nginx
   location ~* \.(glb|gltf|ply)$ {
       expires 30d;
       add_header Cache-Control "public, immutable";
   }
   ```

3. **使用 PM2 集群模式**
   ```bash
   pm2 start server.js -i max --name "3d-model-viewer"
   ```

### 备份策略

# 定期备份数据
tar -czf backup-$(date +%Y%m%d).tar.gz /root/3d-model-viewer/data

# 自动备份脚本
echo "0 3 * * * tar -czf /root/backups/backup-\$(date +\%Y\%m\%d).tar.gz /root/3d-model-viewer/data" | crontab -
