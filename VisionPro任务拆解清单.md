# VisionPro 任务拆解清单

本文档面向项目执行和分工，适合：

- 学生团队按周推进
- 老师或负责人分配任务
- 用作阶段验收清单

如果需要完整背景说明，请配合阅读：

- `VisionPro需求说明文档.md`
- `VisionPro开发入门手册.md`

---

## 1. 使用方式

建议按下面顺序使用本清单：

1. 先完成“基础可运行”
2. 再完成“普通浏览模式稳定”
3. 再完成“后台与数据稳定”
4. 再完成“Vision Pro 链路打通”
5. 最后推进“XR 体验优化”和“架构升级”

---

## 2. 总体目标

本项目当前阶段的总体目标分为 5 类：

1. 系统可运行
2. 普通浏览模式稳定可用
3. 后台和数据链路稳定
4. Vision Pro 可稳定访问并进入 XR
5. 架构升级与效果优化

---

## 3. 里程碑清单

### 里程碑 M1：本地可运行

目标：

- 所有成员都能在自己的电脑上跑起来

完成标准：

- `npm install` 成功
- `npm run cert` 成功
- `npm run dev` 成功
- 可访问前台
- 可访问后台

负责人建议：

- 全员

---

### 里程碑 M2：普通浏览模式可用

目标：

- 普通浏览模式完成基础展示闭环

完成标准：

- 至少一个 `GLB` 模型可显示
- 至少一个 `PLY` 或 `KSPLAT` 模型可显示
- 可通过缩略图切换模型
- 可重置视角
- 可进入全屏
- 模型切换不白屏、不崩溃

负责人建议：

- 前台同学

---

### 里程碑 M3：后台可用

目标：

- 后台完成上传、编辑、删除、顺序调整闭环

完成标准：

- 可上传模型
- 可上传背景图
- 可编辑模型名称和描述
- 可删除模型
- 可删除背景图
- 可调整模型顺序
- 前台刷新后能看到更新结果

负责人建议：

- 后台同学

---

### 里程碑 M4：Vision Pro 访问链路打通

目标：

- Vision Pro 可稳定打开项目并进入 XR

完成标准：

- Vision Pro 能通过 HTTPS 打开页面
- `rootCA.cer` 已安装并信任
- 页面上的 VR/MR 按钮状态正确
- 可成功进入至少一种 XR 模式

负责人建议：

- XR 同学

---

### 里程碑 M5：XR 基础交互可用

目标：

- 完成 VR/MR 的基础浏览和模型操作

完成标准：

- 可进入 VR
- 可进入 MR
- 画廊模式可显示模型
- 可进入详情模式
- 可返回
- 可单手移动
- 可双手缩放旋转

负责人建议：

- XR 同学

---

### 里程碑 M6：中长期优化启动

目标：

- 把项目从“能跑”推进到“更稳定、更好看、更易维护”

完成标准：

- 已启动架构升级评估
- 已梳理普通浏览模式粒子加载 demo 的可迁移内容
- 已形成下一阶段改造方案

负责人建议：

- 组长 + 各模块负责人

---

## 4. 任务池 A：基础环境与启动

### A1. 安装并启动项目

内容：

- 安装依赖
- 生成证书
- 本地启动服务

涉及文件：

- `package.json`
- `generate-mkcert.js`
- `generate-cert.js`
- `server.js`

验收标准：

- 本地能成功启动 HTTPS 服务

优先级：

- 最高

---

### A2. 记录每位同学的本机访问地址

内容：

- 确认每位同学电脑当前局域网 IP
- 更新证书配置

涉及文件：

- `cert-config.json`

验收标准：

- 每位同学都能用自己的 IP 在同网设备上访问

优先级：

- 高

---

### A3. 整理启动说明

内容：

- 统一启动命令
- 写清楚前台地址、后台地址、Vision Pro 地址

验收标准：

- 新同学在不问人的情况下可按文档启动项目

优先级：

- 高

---

## 5. 任务池 B：普通浏览模式

### B1. 普通网格模型显示稳定性

内容：

- 检查 `GLB/GLTF` 加载
- 检查 `OBJ/STL/FBX` 加载
- 检查缩放、朝向、居中是否合理

涉及文件：

- `js/app.js`

验收标准：

- 各类普通网格模型能正常显示

优先级：

- 高

---

### B2. 高斯模型显示稳定性

内容：

- 检查 `PLY/SPLAT/KSPLAT/SPZ` 加载
- 检查切换模型时资源清理
- 检查加载进度显示

涉及文件：

- `js/app.js`
- `js/plyViewer.js`

验收标准：

- 至少一种高斯模型在普通模式下稳定显示

优先级：

- 高

---

### B3. 普通浏览模式交互优化

内容：

- 缩略图切换体验优化
- 加载提示优化
- 错误提示优化
- 空状态页面优化

涉及文件：

- `index.html`
- `js/app.js`
- `css/style.css`

验收标准：

- 用户能看懂系统当前状态，不会遇到“无反馈”

优先级：

- 中高

---

### B4. 普通浏览模式粒子加载效果对齐 demo

内容：

- 阅读现有 demo
- 记录 demo 中可复用的效果点
- 对齐普通浏览模式的加载过渡

重点关注：

- 模型出现方式
- 粒子化加载过渡
- 模型切换视觉反馈
- 整体展示感

验收标准：

- 本项目普通浏览模式在视觉效果上明显向 demo 靠拢

优先级：

- 中高

备注：

- 这是新增目标，不属于可忽略项

---

### B5. 普通模式验收测试

内容：

- 连续切换模型
- 测试全屏
- 测试重置视角
- 测试背景图切换

验收标准：

- 不出现明显崩溃、黑屏、模型残留

优先级：

- 高

---

## 6. 任务池 C：后台与数据

### C1. 模型上传流程核查

内容：

- 检查后台上传模型是否成功

涉及文件：

验收标准：

- 上传一个模型后，前后台都能正确看到

优先级：

- 高

---

### C2. 背景图上传与绑定流程核查

内容：

- 上传背景图
- 给模型绑定背景图
- 检查前台是否显示对应背景

验收标准：

- 模型与背景图绑定关系生效

优先级：

- 中高

---

### C3. 删除逻辑清理

内容：

- 删除模型时清理文件
- 删除缩略图时清理旧文件
- 删除背景图时检查引用残留

验收标准：

- 删除后数据和文件保持一致

优先级：

- 高

---

### C4. 上传校验加强

内容：

- 校验文件类型
- 校验文件大小
- 校验空字段
- 优化错误提示

验收标准：

- 非法文件不能被静默写入

优先级：

- 高

---

### C5. 后台格式支持统一

内容：

- 对齐“代码支持格式”和“后台可上传格式”

当前不一致点：

- 前台代码支持 `obj/stl/fbx`
- 后台上传入口未开放

验收标准：

- 文档、前台能力、后台入口三者一致

优先级：

- 中高

---

### C6. 批量删除功能补全

内容：

- 补齐更多格式选项
- 保证前后端能力一致

验收标准：

- 批量删除功能和服务端真实能力一致

优先级：

- 中

## 7. 任务池 D：Vision Pro 与证书链路

### D1. 证书链路跑通

内容：

- 确认 `mkcert` 证书生成成功
- 确认 `rootCA.cer` 可安装
- 确认设备可信任证书

涉及文件：

- `generate-mkcert.js`
- `cert-config.json`
- `cert/`

验收标准：

- Vision Pro 可无阻塞访问 HTTPS 页面

优先级：

- 最高

---

### D2. 清理硬编码 IP

内容：

- 梳理代码和文档中的固定 IP
- 改为更通用的配置方式

涉及文件：

- `cert-config.json`
- `generate-mkcert.js`
- `server.js`

验收标准：

- 换网络时不需要到处手改代码

优先级：

- 高

---

### D3. 访问排查手册

内容：

- 写一页简单排查说明

至少覆盖：

- 页面打不开
- 证书不可信
- XR 按钮不可用
- 设备访问不到开发机

验收标准：

- 新同学遇到 Vision Pro 访问问题能自己排查

优先级：

- 中高
